# アウトラインテンプレート

スライドデッキのアウトラインの標準構造。

## フォーマット

```markdown
# スライドデッキ アウトライン

**トピック**: [トピックの説明]
**カラーセット**: [カラーセットID]
**スライド数**: N
**言語**: [出力言語]

---

## スライド1 — カバー
- **レイアウト**: title-hero
- **背景**: gradient
- **見出し**: [メインタイトル — 物語型、印象的に]
- **サブタイトル**: [補足のタグラインまたは文脈]

---

## スライド2 — [説明的な名前]
- **レイアウト**: title-body
- **タイトル**: [物語型の見出し — ストーリーを語る]
- **本文**:
  - [具体的な詳細を含むポイント1]
  - [具体的な詳細を含むポイント2]
  - [具体的な詳細を含むポイント3]
- **トランジション**: fade

---

## スライドN — クロージング
- **レイアウト**: closing
- **メッセージ**: [印象的な締めの一言またはコールトゥアクション]
- **サブテキスト**: [次のステップまたはまとめ]
- **背景**: gradient | color
```

## スライドエントリーのフィールド

| フィールド | 必須 | 説明 |
|-----------|------|------|
| レイアウト | はい | layouts.md のレイアウトパターン名 |
| 背景 | カバー/区切りのみ | `gradient` または `color`（デフォルトはcolor） |
| タイトル/見出し | はい | 物語型の見出し（ラベルではない） |
| サブタイトル | カバーのみ | 補足テキスト |
| 本文 | コンテンツスライド | 箇条書きまたは段落 |
| トランジション | いいえ | デフォルト（fade）の上書き |

## アウトライン前のコンテンツ分析

アウトラインを書く前に以下の分析を行う:

### 1. 核心メッセージ
- 最も重要な要点はひとつだけ
- 15語以内で述べる

### 2. サポートポイント（最大3〜5個）
- 核心メッセージを裏付ける根拠は何か？
- 聴衆にとっての関連度で優先順位をつける

### 3. ビジュアルの機会

| コンテンツタイプ | レイアウト |
|----------------|----------|
| 比較 | `comparison` または `two-column` |
| プロセス/ステップ | `title-body`（番号付き項目） |
| 重要な統計 | `key-stat` |
| 引用/洞察 | `quote` |
| 3つの概念 | `three-point` |

### 4. 構成パターン
コンテンツに合わせて選択:
- 課題 → 解決策 → メリット
- 何を → なぜ → どうやって
- 過去 → 現在 → 未来
- 文脈 → 根拠 → アクション

## スライドタイプ別テンプレート

### カバー
```
## スライド1 — カバー
- **レイアウト**: title-hero
- **背景**: gradient
- **見出し**: [印象的なタイトル]
- **サブタイトル**: [文脈またはタグライン]
```

### コンテンツ（標準）
```
## スライドX — [名前]
- **レイアウト**: title-body
- **タイトル**: [物語型の見出し]
- **本文**:
  - [具体的なポイント1]
  - [具体的なポイント2]
  - [具体的なポイント3]
```

### セクション区切り
```
## スライドX — セクション: [名前]
- **レイアウト**: section-divider
- **背景**: gradient
- **セクション名**: [セクション名]
```

### 統計
```
## スライドX — [名前]
- **レイアウト**: key-stat
- **数値**: [単位付きの大きな統計値]
- **ラベル**: [数値が表すもの]
- **文脈**: [なぜ重要か]
```

### 引用
```
## スライドX — [名前]
- **レイアウト**: quote
- **引用**: [引用テキスト]
- **出典**: [ソース]
```

### 比較
```
## スライドX — [名前]
- **レイアウト**: comparison
- **タイトル**: [比較のフレーミング]
- **左ラベル**: [カテゴリA]
- **左ポイント**: [項目]
- **右ラベル**: [カテゴリB]
- **右ポイント**: [項目]
```

### クロージング
```
## スライドN — クロージング
- **レイアウト**: closing
- **メッセージ**: [インパクトのある締め]
- **サブテキスト**: [コールトゥアクションまたは次のステップ]
```

## スライド数ガイドライン

| コンテンツ量 | 推奨枚数 | 構成 |
|-------------|---------|------|
| 短い（500語未満） | 5-8 | カバー + 3-5コンテンツ + クロージング |
| 中程度（500-2000語） | 8-15 | カバー + 1-2セクション + クロージング |
| 長い（2000-5000語） | 12-20 | カバー + 2-3セクション + クロージング |
| 非常に長い（5000語超） | 15-25 | カバー + 3-4セクション + クロージング |

**セクション構成**: セクション区切り + 3〜5枚のコンテンツスライド/セクション。
