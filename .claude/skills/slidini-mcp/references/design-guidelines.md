# デザインガイドライン

スライド生成におけるビジュアル階層とデザイン原則。

## キャンバス仕様

| プロパティ | 値 |
|-----------|------|
| 幅 | 1920 px |
| 高さ | 1080 px |
| アスペクト比 | 16:9 |
| マージン（水平） | 端から100px |
| マージン（垂直） | 端から50px |
| セーフエリア | 100-1820 (x), 50-1030 (y) |

## 基本原則

### 1. キャンバスを埋める

**余白を無駄にしない。** コンテンツ領域はキャンバスを最大限に活用する。

- タイトルは y=50 から開始し、上部の無駄な空白を排除
- ボディは y=220 からタイトル直下に配置し、下端 y=1000 付近まで使う
- 左右マージンは 100px に抑え、コンテンツ幅を最大化

### 2. 読みやすい文字サイズ

**遠くからでも読める文字サイズ。** プレゼンテーションは大画面で表示される前提。

- ボディテキストは最低 34px
- 箇条書きは最低 32px
- 28px 未満のテキストは使わない（キャプション・属性表記のみ例外）

### 3. 文字の見切れ防止

**文字の見切れは絶対に防ぐ。** 要素の height はコンテンツ量に応じて必ず調整する。

```
必要な高さ = 行数 x fontSize x lineHeight + padding x 2 + 10
```

- **`padding x 2`**: `style.padding` は上下両方に適用される。バッジ（`backgroundColor` 付きテキスト）では特に重要
- Markdown の箇条書きは1項目 = 最低1行。長い項目は折り返し行数も加算
- コードブロックの各行もカウント対象

例: padding=14, fontSize=32, lineHeight=1.5 の1行テキスト:
```
1行 x 32px x 1.5 + 14 x 2 + 10 = 86px → height: 86 以上
```

例: padding=0, fontSize=36, lineHeight=1.7 の5項目箇条書き（各2行）:
```
5項目 x 2行 x 36px x 1.7 + 0 + 10 = 622px → height: 650 以上
```

## 見切れ修正の優先順序

要素の見切れが発生した場合、以下の順で対処する:

1. **height 拡大**: 下端が accentBar（y:1070）の10px手前に収まる範囲で拡大
2. **y 微調整**: 上の要素との間隔を10px以上確保しつつ上に移動
3. **fontSize / lineHeight 調整**: フォントサイズを2px刻みで縮小
4. **コンテンツ分割**: 箇条書き5項目以上、本文350文字以上なら2スライドに分割

### レイアウト安全チェック

要素配置後に以下を検証:

```
要素.top >= 前の要素.bottom + 10px    # 要素間に最低10pxの隙間
要素.bottom <= accentBarY - 10px       # アクセントバー手前で収まる
要素.bottom <= 1060                    # バーがない場合もキャンバス内
```

## オーバーレイとの重なり

前面オーバーレイ（キャラクター画像など）がある場合の要素配置ルール:

### ブロック要素は重ねてOK
- `backgroundColor` 付きのカード、リスト項目の背景などのブロック要素はオーバーレイと重なっても問題ない
- **幅を狭めてオーバーレイを避けない** — 左右非対称になり見た目が悪い
- 要素の `width` は常にフル幅（1720px）を維持する

### テキストの改行で回避
- オーバーレイと重なるエリアでは、テキスト内容が自然に手前で折り返されるようにする
- コンテンツ末尾の行が短くなるよう、改行位置を調整する
- 1行あたりの文字数を意識し、最終行がオーバーレイ領域に到達しないようにする

### アイコン配置
- スライドタイトル横のアイコンは、タイトルテキスト幅を推定して直後（+20px gap）に配置
- 右上固定配置はオーバーレイのタイトルバーと被るため避ける

## タイトルスライド装飾パターン

カバースライドやセクション区切りの背景装飾として、以下のプリセットが使える。

### Fractured Data Panels（テック・AI系推奨）

異なるサイズ・角度・opacity の半透明パネルを重ねて「データの層」を表現する。

**構成要素（zIndex: 0 で背面配置）:**

| 種別 | 数 | パラメータ範囲 |
|------|---|---------------|
| パネル（大） | 3〜5枚 | width:340-550, height:150-210, opacity:0.12-0.22, rotation:±15° |
| 縦線 | 1〜2本 | width:2-3, height:400-540, opacity:0.20-0.35, rotation:±10° |
| 横線 | 1〜2本 | width:120-640, height:2-6, opacity:0.15-0.35, rotation:±5° |

**カラー:**
- パネル: 暗い青系（`#1e3a8a`, `#312e81`, `#2563eb`）や明るい青（`#0ea5e9`）
- 線: シアン系（`#67e8f9`, `#22d3ee`, `#60a5fa`, `#93c5fd`）

**配置エリア:** x:1100〜1800, y:100〜700（右半分に集中）

**注意:** コンテンツスライドには使わない。タイトル・セクション区切り専用。

## ビジュアル階層

| 原則 | 説明 |
|------|------|
| フォーカルポイント | 1スライドにつき1つの支配的要素が最初に目を引く |
| サイズコントラスト | 見出しは本文の1.5〜2倍の大きさ |
| 余白の確保 | 水平端から最低100px、垂直端から50pxのマージン |
| 揃え | コンテンツスライドは左揃え統一、タイトルスライドは中央揃え |
| Z-Index | 前面に表示すべき要素はz-indexを高く |

## カラー適用

### カラーセットのロールマッピング

| ロール | 用途 | 典型的な要素 |
|--------|------|-------------|
| `background` | スライド背景 | スライドの背景色 |
| `surface` | カード、アクセント背景 | テキスト要素のbackgroundColor |
| `textPrimary` | 見出し、メインテキスト | タイトルの文字色 |
| `textSecondary` | 本文、箇条書き | 本文の文字色 |
| `textMuted` | キャプション、属性表記 | 小さい/補足テキストの色 |
| `accent` | 強調、プライマリハイライト | セクションヘッダー背景、グラデーション開始色、統計数値 |
| `accentSecondary` | セカンダリハイライト | 比較ヘッダー背景、グラデーション終了色 |

### 背景の使い分け

| スライドタイプ | 背景 |
|--------------|------|
| カバー | グラデーション（accent → accentSecondary） |
| コンテンツ | ソリッド（background色） |
| セクション区切り | グラデーション（accent → accentSecondary） |
| 引用 | ソリッド（background色） |
| 統計 | ソリッド（background色）またはsurface色 |
| クロージング | グラデーション（カバーと統一）またはソリッド |

| レベル | fontSize | fontWeight | カラーロール | 用途 |
|--------|----------|------------|-------------|------|
| ディスプレイ | 80 | bold | textPrimary | カバータイトルのみ |
| H1 | 64 | bold | textPrimary | セクション区切り |
| H2 | 52 | bold | textPrimary | スライドタイトル |
| H3 | 40 | bold | textPrimary | サブタイトル、小見出し |
| 本文 | 36 | normal | textSecondary | メインの本文 |
| 小テキスト | 32 | normal | textSecondary | 箇条書き、カラムテキスト |
| キャプション | 28 | normal | textMuted | 属性表記、注釈 |
| 統計数値 | 140-180 | bold | accent | 大きな統計値 |

## アニメーションガイドライン

### アニメーションを付ける場面

| 要素タイプ | アニメーション | 推奨度 |
|-----------|-------------|--------|
| カバータイトル | fade-in | 常に付ける |
| カバーサブタイトル | fade-in（遅延あり） | 常に付ける |
| スライドタイトル | なし | デフォルトはなし |
| 本文コンテンツ | fade-in | 任意、強調したい場合 |
| 統計数値 | scale-in | 推奨 |
| 引用テキスト | fade-in | 推奨 |

### アニメーションのデフォルト値

```
duration: 0.5秒
delay: 0.2秒（連続する要素は0.15秒ずつ加算）
easing: ease-out
trigger: onEnter
stepIndex: 0（常に表示）
```

### 段階的表示

`stepIndex > 0` は控えめに使用:
- OK: title-bodyスライドで3つの要点を1つずつ表示
- OK: 比較カラムを順番に表示
- NG: すべてのスライドのすべての要素にstepIndexを設定

## トランジションガイドライン

| コンテキスト | トランジション | 所要時間 |
|-------------|-------------|---------|
| デフォルト | `fade` | 0.5秒 |
| セクション切替 | `fade` | 0.5秒 |
| 順方向の流れ | `slide-left` | 0.5秒 |
| 強調の場面 | `zoom` | 0.4秒 |
| カバー → コンテンツ | `fade` | 0.6秒 |

## 一貫性ルール

| 要素 | ガイドライン |
|------|-------------|
| マージン | すべてのスライドで水平100px、垂直50pxを統一 |
| タイトル位置 | すべてのtitle-bodyスライドで同じx,y位置 |
| フォントサイズ | 同じコンテンツタイプは全スライドで同じサイズ |
| カラー | 選択したカラーセットの色のみ使用 |
| トランジション | すべてのコンテンツスライドで同じトランジションタイプを使用 |

## スライドタイプ別コンテンツ密度

| タイプ | 最大テキスト要素数 | 最大箇条書き数 | 備考 |
|--------|-------------------|---------------|------|
| カバー | 2（タイトル + サブタイトル） | 0 | ミニマルに |
| コンテンツ | 2（タイトル + 本文） | 5 | 各項目2〜3行で詳細を含む |
| 2カラム | 3（タイトル + 2カラム） | 各カラム4 | 両側のバランスを取る |
| 比較 | 5（タイトル + 2ヘッダー + 2本文） | 各サイド4 | 並列構造を維持 |
| セクション区切り | 1 | 0 | セクション名のみ |
| 引用 | 2（引用 + 出典） | 0 | 引用に余白を持たせる |
| 統計 | 3（数値 + ラベル + 文脈） | 0 | 数値を支配的に |
| クロージング | 2（メッセージ + サブテキスト） | 0 | インパクトがありクリーンに |
