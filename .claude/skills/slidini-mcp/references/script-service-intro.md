# サービス紹介向け台本ガイド

## ターゲット視聴者

サービスの見込みユーザー、既存ユーザー、メディア関係者など。技術的なバックグラウンドは想定しない（ユーザーが技術者向けと指定した場合は `web-engineer.md` と併用）。視聴者が「このサービスは自分に合っているか」を判断できる情報を提供することがゴール。

## トーン

「です・ます」調で、わかりやすさと信頼感を重視する。「プロダクトを愛しているチームメンバーが、使い方を教えてくれている」ような雰囲気を目指す。機能の羅列ではなく、ユーザー体験を軸にした語りが効果的。

## フックの切り口

サービス紹介の視聴者は「自分の課題を解決できるか」を知りたくて見始める。以下のようなパターンが効果的：

- **課題共感型**: 「○○って、面倒ですよね」「○○にかかる時間、もっと短くしたいと思いませんか？」
- **デモ先行型**: 「まず、実際の画面を見てください」（画面キャプチャ指示で始める）
- **ビフォーアフター型**: 「○○が、たった□ステップで△△に変わります」
- **ユーザーの声型**: 「○○を使い始めてから、□□が劇的に改善しました」

## 本題の構成パターン

サービス紹介では「何ができるか → どう使うか → なぜ選ぶべきか」の流れが効果的：

### 新規ユーザー向けパターン
1. 解決する課題（ペインポイント）
2. サービスの概要（一言で言うと何か）
3. 主要機能のデモ（3〜5機能に絞る）
4. 他サービスとの違い
5. 料金プランの概要
6. 始め方（導入ステップ）

### 機能紹介・アップデート告知パターン
1. 新機能の概要と背景
2. 具体的な使い方デモ
3. 活用シーン・ユースケース
4. 既存機能との組み合わせ
5. 利用開始方法

### 導入事例パターン
1. 導入企業の課題
2. なぜこのサービスを選んだか
3. 導入プロセス
4. 効果・成果（数値で示す）
5. 今後の活用予定

### 構成上の注意点

- 機能は絞る。全機能を網羅しようとすると「結局何がすごいのか」が伝わらない。3〜5つの主要機能に集中する
- 各機能は「機能名」ではなく「ユーザーが得られる価値」で紹介する。「タスク管理機能」→「チームの進捗が一目でわかる」
- デモ的な要素（画面キャプチャ、操作手順）を入れると理解度が大幅に上がる

## ビジュアル要素

共通のビジュアル要素に加え、以下を積極的に活用する：

### 画面キャプチャ・UI指示

サービスの実際の画面を見せるのが最も効果的。「○○画面のスクリーンショットを挿入」「ダッシュボード画面を表示」のように具体的に指示する。

### 操作フロー

ユーザーの操作手順をステップバイステップで見せる。
例：「① ログイン → ② プロジェクト作成 → ③ メンバー招待 → ④ タスク追加」

### ユースケース図

「誰が」「どんな場面で」「どう使うか」を図解する。ペルソナ別の使い方を示すと、幅広い視聴者に刺さる。

### 料金比較表

プランの違いを一目で比較できる表。無料プランがある場合は特に強調する。

| | Free | Pro | Enterprise |
|---|---|---|---|
| ユーザー数 | 3名まで | 無制限 | 無制限 |
| ストレージ | 1GB | 100GB | 無制限 |
| サポート | メール | チャット | 専任担当 |

### 導入効果の数値

導入事例がある場合、効果を数字で見せるスライドは説得力が高い。
例：「作業時間 60%削減 / エラー率 90%減少 / 顧客満足度 +25pt」

## まとめでの行動喚起

- 無料トライアル・フリープランの開始
- デモリクエスト
- 概要欄のリンクからサインアップ
- 公式ドキュメント・ヘルプセンターの確認

## ファクトチェックの重点項目

サービス紹介では最新性と正確性が特に重要：

- 機能の説明が最新のバージョンに基づいているか
- 料金プランが現行のものか（頻繁に変わるため要注意）
- 導入効果の数値に出典があるか
- 競合との比較が公平か（機能の有無だけでなく条件を揃えているか）
- 対応プラットフォームやブラウザの情報が最新か
- 連携サービス一覧が正しいか

## 留意事項

- 機能の説明は「What（何ができるか）」だけでなく「So What（だから何が嬉しいか）」まで踏み込む
- 料金は頻繁に変わるため、動画内では大枠を示し、詳細は概要欄のリンクに誘導するのが運用しやすい
- 競合サービス名を直接出すかどうかはユーザーに確認する。出す場合も攻撃的な比較は避ける
- 「誰にでも合うサービス」という見せ方より、「こういう人・チームに最適」と明確にしたほうが結果的に刺さる
